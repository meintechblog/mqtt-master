<task type="auto">
  <name>Task 1: Create install.sh installer script</name>
  <files>install.sh</files>
  <action>
Create `install.sh` in the repo root. The script must be idempotent -- safe to run on fresh systems and existing installs.

**Script structure:**

1. **Preamble**: `#!/bin/bash`, `set -e`, root check (`if [ "$(id -u)" -ne 0 ]; then echo "Run as root"; exit 1; fi`), detect OS (must be Debian or Ubuntu via /etc/os-release).

2. **Node.js check/install**: Check if `node` exists and is v20+. If missing or too old, install from NodeSource:
   ```
   curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
   apt-get install -y nodejs
   ```

3. **Mosquitto check/install**: `apt-get install -y mosquitto`. Write config ONLY if `/etc/mosquitto/conf.d/mqtt-master.conf` does not exist:
   ```
   listener 1883
   allow_anonymous true
   ```
   Then `systemctl enable mosquitto && systemctl restart mosquitto`.

4. **Git install**: `apt-get install -y git` if not present.

5. **Application install/update**:
   - If `/opt/mqtt-master/.git` exists (update mode): `cd /opt/mqtt-master && git pull`
   - Else (fresh install): `git clone https://github.com/meintechblog/mqtt-master.git /opt/mqtt-master`
   - Run `cd /opt/mqtt-master && npm install --production`

6. **Config preservation**: If `/opt/mqtt-master/config.json` does not exist, create default:
   ```json
   {
     "mqtt": { "broker": "mqtt://localhost:1883" },
     "web": { "port": 3000 },
     "logLevel": "info"
   }
   ```
   If it already exists, do NOT touch it.

7. **systemd service**: Write `/etc/systemd/system/mqtt-master.service`:
   ```ini
   [Unit]
   Description=MQTT Master
   After=network.target mosquitto.service

   [Service]
   Type=simple
   WorkingDirectory=/opt/mqtt-master
   ExecStart=/usr/bin/node server/index.js
   Restart=always
   RestartSec=5
   Environment=NODE_ENV=production

   [Install]
   WantedBy=multi-user.target
   ```
   Then `systemctl daemon-reload && systemctl enable mqtt-master && systemctl restart mqtt-master`.

8. **Success message**: Print the URL (`http://<hostname>:3000`) and status. Use `hostname -I | awk '{print $1}'` for IP.

Mark the file executable: the script itself should note `chmod +x` is not needed when piped via `wget | bash`, but the file in repo should have execute permission set via `git update-index --chmod=+x install.sh` or equivalent.

**Important details:**
- Use `apt-get` not `apt` (non-interactive compatibility)
- Add `DEBIAN_FRONTEND=noninteractive` export at top
- All apt commands use `-y` flag
- Print clear progress messages for each step (no emojis, plain text labels like "Installing Node.js...", "Configuring Mosquitto...", etc.)
- The script must work when piped: `wget -qO- URL | bash` and also when downloaded and run: `bash install.sh`
