       - `reloadPlugin(id)` -- POST /api/plugins/:id/reload.
       - `getPluginConfig(id)` -- GET /api/plugins/:id/config, returns { config, schema }.
       - `savePluginConfig(id, config)` -- PUT /api/plugins/:id/config with JSON body.

       - In the Sidebar component, add a `useEffect` that calls `fetchPlugins()` on mount, maps each plugin to `{ label: plugin.name || plugin.id, hash: '#/plugins/' + plugin.id, status: plugin.status }`. Store in `pluginItems.value`.
       - Set up a 5-second polling interval inside the same useEffect to refresh plugin statuses. Clear interval on cleanup.
       - Replace the static `sections` array: the Plugins section's items should read from `pluginItems.value`.
       - Each plugin nav item should show a StatusDot next to the label. Map plugin status: 'running' -> 'connected', 'error' -> 'error', 'stopped' -> 'stopped'. Render StatusDot inline before the label text within the nav item link.
       - Remove the `.filter(s => s.items.length > 0)` so the PLUGINS section always shows (even if empty, showing "No plugins" text).

    4. Create `webapp/js/pages/plugin-config.js` -- the PluginConfig page component:
       - Receives `pluginId` prop (extracted from hash).
       - On mount, calls `getPluginConfig(pluginId)` to get `{ config, schema }`.
       - Also calls `fetchPlugins()` and finds current plugin to get status.
       - Renders page header with plugin name and StatusDot.
       - Renders action buttons: Start (disabled if running), Stop (disabled if stopped), Reload (always enabled). Each calls the corresponding API function, then refreshes status.
       - Renders auto-generated form from JSON Schema:
         - Iterate `schema.properties`. For each property:
           - `type: 'string'` -> text input

           - `type: 'boolean'` -> checkbox
         - Use property key as field name, `title` as label (fallback to key), `default` as placeholder.
         - Form state managed with `useSignal` or local state (signal per field or one signal holding config object).

       - Save button calls `savePluginConfig(pluginId, formData)`.
       - Show success/error feedback after save (brief text message, auto-dismiss after 3s).
       - Use Venus OS theme classes: `.ve-card` for form container, `.ve-panel` for button group, standard input styles matching `.msg-topic-input`.

       - Update the `currentPage` computed signal to handle this pattern. Since the route map is static, add a computed that first checks the static routes map, then checks the `#/plugins/` prefix pattern, then falls back to NotFound.
