<task type="auto" tdd="true">
  <name>Task 1: PluginManager service + example plugin + ConfigService.set/save</name>
  <files>server/services/plugin-manager.js, server/services/config-service.js, plugins/example/plugin.js, tests/plugin-manager.test.js</files>
  <behavior>
    - Test: discover() finds plugin directories that contain plugin.js
    - Test: start(id) calls plugin.start(context) and status becomes 'running'
    - Test: stop(id) calls plugin.stop() and status becomes 'stopped'
    - Test: reload(id) stops then re-imports and starts the plugin
    - Test: start(id) on unknown plugin throws/returns error
    - Test: plugin crash in start() sets status to 'error' with error message
    - Test: getConfig(id) returns config from configService under plugins.<id>
    - Test: setConfig(id, data) persists config via configService
    - Test: listAll() returns array of {id, name, status, error?}
  </behavior>
  <action>
    1. Add `set(key, value)` and `async save()` methods to ConfigService (config-service.js). `set` writes a dot-notation key into the config object. `save` writes config to disk with `writeFile` + JSON.stringify with 2-space indent.

    2. Create `server/services/plugin-manager.js` exporting class PluginManager:
       - Constructor takes `{ pluginDir, configService, mqttService, logger }`.
       - `async discover()` — reads pluginDir with `readdir`, filters for dirs containing `plugin.js`. Stores metadata in `this.plugins` Map keyed by directory name (plugin ID). Each entry: `{ id, name, status: 'stopped', instance: null, error: null, modulePath }`.
       - `async start(id)` — dynamic `import(modulePath)` to get plugin class (default export), instantiate, call `instance.start({ mqttService, configService, logger, pluginId: id })`. Wrap in try/catch; on error set status='error' and store error message. On success set status='running'.
       - `async stop(id)` — call `instance.stop()`, set status='stopped', null instance. Wrap in try/catch.
       - `async reload(id)` — stop if running, delete from Node's module cache by appending `?t=Date.now()` to import URL for cache-busting, then start.
       - `listAll()` — returns array of `{ id, name, status, error }` from this.plugins Map.
       - `getStatus(id)` — returns `{ status, error }` for single plugin.
       - `getConfig(id)` — returns `configService.get('plugins.' + id, {})`.
       - `async setConfig(id, data)` — calls `configService.set('plugins.' + id, data)` then `configService.save()`.

    3. Create `plugins/example/plugin.js` — a minimal example plugin:
       - Default export: class ExamplePlugin with `start(context)`, `stop()`, `getStatus()`, `getConfigSchema()`.
       - getConfigSchema returns: `{ type: 'object', properties: { greeting: { type: 'string', title: 'Greeting', default: 'Hello' } } }`.
       - start() logs "Example plugin started" via context.logger, stores context reference.
       - stop() logs "Example plugin stopped".
       - getStatus() returns { running: true/false }.

    4. Write tests in `tests/plugin-manager.test.js` using vitest. Use a temp directory with a mock plugin for discover/start/stop/reload tests. Mock configService and mqttService. Test error handling for plugin crash scenarios.
  </action>

  </verify>

    - Test: POST /api/plugins/example/start returns 200 and starts plugin
    - Test: POST /api/plugins/example/stop returns 200 and stops plugin
    - Test: POST /api/plugins/example/reload returns 200
    - Test: GET /api/plugins/example/config returns config object + schema
    - Test: PUT /api/plugins/example/config saves config and returns 200
    - Test: POST /api/plugins/nonexistent/start returns 404
  </behavior>
  <action>
    1. Create `server/routes/api-plugins.js` as a Fastify plugin (same pattern as ws-dashboard.js / ws-messages.js):
       - `GET /api/plugins` — calls `app.pluginManager.listAll()`, returns JSON array.
       - `POST /api/plugins/:id/start` — calls `app.pluginManager.start(id)`, returns `{ ok: true }` or `{ error }` with 404 if plugin not found.
       - `POST /api/plugins/:id/stop` — calls `app.pluginManager.stop(id)`, returns `{ ok: true }`.
       - `POST /api/plugins/:id/reload` — calls `app.pluginManager.reload(id)`, returns `{ ok: true }`.
       - `GET /api/plugins/:id/config` — returns `{ config: ..., schema: ... }`. Gets config from pluginManager.getConfig(id), schema from plugin instance's getConfigSchema() (or empty object if not running).
       - `PUT /api/plugins/:id/config` — calls `app.pluginManager.setConfig(id, request.body)`, returns `{ ok: true }`.
       - All routes wrap in try/catch returning 500 with `{ error: message }` on failure.
       - Return 404 with `{ error: 'Plugin not found' }` when plugin ID doesn't exist in manager.

    2. Wire into `server/index.js`:
       - Import PluginManager and api-plugins route.
       - After existing service setup, create PluginManager instance: `new PluginManager({ pluginDir: config.get('pluginDir', join(__dirname, '..', 'plugins')), configService: config, mqttService, logger: app.log })`.
       - Call `await pluginManager.discover()`.
       - Decorate: `app.decorate('pluginManager', pluginManager)`.
       - Register route: `await app.register(apiPlugins)`.
       - Place BEFORE the SPA fallback handler so /api/plugins routes are matched.

    3. Write integration tests in `tests/api-plugins.test.js`. Spin up Fastify with mock mqttService, real ConfigService (temp file), register api-plugins route with a mock PluginManager. Test each endpoint's response shape and status codes.
