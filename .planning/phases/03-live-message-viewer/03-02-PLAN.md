  <name>Task 1: WebSocket messages client module and Messages page UI</name>
  <files>webapp/js/lib/ws-messages-client.js, webapp/js/pages/messages.js, webapp/css/theme.css</files>
  <action>
    1. Create webapp/js/lib/ws-messages-client.js following the ws-client.js singleton pattern:
       - Signals: `messages` (signal, array of {topic, payload, timestamp}), `subscriptions` (signal, Set of active topic strings), `messagesWsConnected` (signal, boolean)
       - `connectMessagesWs()` -- connects to `/ws/messages` WebSocket with same exponential backoff pattern as ws-client.js
       - On message from server:
         - type "message": prepend to messages array, enforce 500 max ring buffer (slice if over), increment a rate counter variable
         - type "subscribed": add topic to subscriptions signal Set
         - type "unsubscribed": remove topic from subscriptions signal Set
         - type "error": console.warn the error message
       - `subscribeTopic(topic)` -- sends {"action":"subscribe","topic"} via WebSocket
       - `unsubscribeTopic(topic)` -- sends {"action":"unsubscribe","topic"} via WebSocket
       - `clearMessages()` -- sets messages signal to empty array
       - `getMessageRate()` -- returns messages/second. Track with a simple counter: increment on each message, every 1000ms read and reset the counter. Expose as a `messageRate` signal updated by setInterval.
       - Do NOT auto-connect on import (unlike dashboard ws-client). Connect when Messages page mounts, disconnect when it unmounts. Export connect/disconnect functions.
       - `disconnectMessagesWs()` -- closes WebSocket, clears reconnect timer, unsubscribes all active topics before closing.

    2. Rewrite webapp/js/pages/messages.js replacing the placeholder:
       - Import { html } from 'htm/preact', { useEffect, useState, useRef } from 'preact/hooks', and all exports from ws-messages-client.js
       - useEffect on mount: call connectMessagesWs(), return cleanup that calls disconnectMessagesWs()
       - **Controls bar** (top of page, inside a `.ve-panel`):
         - Topic input field (text input, placeholder "Topic pattern, e.g. # or home/+/temp")
         - Subscribe button (if topic not in subscriptions) / Unsubscribe button (if topic is in subscriptions). Use `--ve-blue` for subscribe, `--ve-red` for unsubscribe.
         - Clear button (calls clearMessages, secondary style)
         - Message rate display: small text showing "{N} msg/s" using messageRate signal. Use `--ve-green` color when rate > 0.
       - **Active subscriptions** -- small row of pill/tag badges showing each active subscription topic. Each pill has an X button to unsubscribe that topic.
       - **Filter input** -- text input above message list, placeholder "Filter messages..." Filters displayed messages by substring match on topic or payload. Uses local useState, not a signal. Instant filtering.
       - **Message list** (inside a `.ve-card`, scrollable with max-height ~70vh):
         - Each message row: timestamp (HH:MM:SS.mmm format, dim color) | topic (secondary color) | payload (monospace font, primary color)
         - Use a simple div per message row with flexbox, not a table
         - Newest messages at top (they are prepended in the ring buffer)
         - Auto-scroll: the container should stay scrolled to top (showing newest) unless user has scrolled down. Track with an onScroll handler: if scrollTop > 0, set a "userScrolled" flag. When new messages arrive and userScrolled is false, scrollTop stays at 0 (which is natural since we prepend). If user scrolls back to top (scrollTop === 0), reset the flag.
       - **Empty state** -- when no messages and no subscriptions, show helpful text: "Subscribe to a topic pattern to start seeing messages."
       - **Empty state with subscription** -- when subscribed but no messages yet: "Waiting for messages on {topics}..."

    3. Add CSS to webapp/css/theme.css at the end:
       - `.msg-controls` -- flexbox row with gap, align items center, wrap on mobile
       - `.msg-topic-input` -- styled text input matching Venus OS theme (dark bg, light border, light text, rounded)
       - `.msg-btn` -- button base style (padding, border-radius, cursor pointer)
       - `.msg-btn--subscribe` -- blue background
       - `.msg-btn--unsubscribe` -- red background
       - `.msg-btn--clear` -- transparent with border
       - `.msg-rate` -- small text, monospace
       - `.msg-subscriptions` -- flex row of pills
       - `.msg-pill` -- small rounded pill with topic text and X close button
       - `.msg-filter` -- styled text input, full width
       - `.msg-list` -- scrollable container, max-height 70vh, overflow-y auto
       - `.msg-row` -- flex row: timestamp, topic, payload with gaps
       - `.msg-timestamp` -- dim color, monospace, min-width for alignment
       - `.msg-topic` -- secondary color, word-break
       - `.msg-payload` -- monospace, primary color, word-break, flex 1
       - `.msg-empty` -- centered dim text for empty states
       - Responsive: on mobile (max-width 768px), controls bar stacks vertically
