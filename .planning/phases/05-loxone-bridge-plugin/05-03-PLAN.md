  <name>Task 1: Controls API, topic routes backend, and Loxone UI pages</name>
  <files>server/routes/api-plugins.js, plugins/loxone/plugin.js, webapp/js/pages/loxone-controls.js, webapp/js/pages/topic-routes.js, webapp/js/lib/api-client.js, webapp/js/app.js, webapp/js/components/sidebar.js</files>
  <action>
**Backend changes:**

1. **Add controls endpoint to api-plugins.js**: Add `GET /api/plugins/loxone/controls` that returns the control list from the running plugin instance. Access the plugin instance via `app.pluginManager.getInstance('loxone')`, call a new method `getControls()` on it. Returns array of `{ uuid, name, type, room, topic, enabled }`. Also add `PUT /api/plugins/loxone/controls/:uuid` to toggle a control's enabled state (updates `plugins.loxone.disabledControls` array in config).

2. **Add getControls() to LoxonePlugin** (plugins/loxone/plugin.js): Returns `this.structure.getAll()` with each entry augmented with `enabled: !this.disabledControls.includes(uuid)`. Also add `setControlEnabled(uuid, enabled)` method that updates the disabledControls config array and saves.

3. **Add topic route logic to LoxonePlugin**: On `start()`, read `config.topicRoutes` array (default `[]`). Each route: `{ id, sourceTopic, targetTopic, direction: 'inbound'|'outbound', enabled }`.
   - For inbound routes (external -> Loxone): subscribe to sourceTopic on MQTT, on message forward payload to targetTopic (which is a Loxone cmd topic)
   - For outbound routes (Loxone -> external): when a Loxone state is published to a matching sourceTopic, also publish to targetTopic
   - Store active subscriptions so they can be cleaned up on stop()
   - Add `getTopicRoutes()` and `setTopicRoutes(routes)` methods

4. **Add topic routes API endpoints**: `GET /api/plugins/loxone/routes` returns current routes. `PUT /api/plugins/loxone/routes` saves routes array to config and re-applies subscriptions.

**Frontend changes:**

5. **Create `webapp/js/pages/loxone-controls.js`**: Preact component `LoxoneControls` that:
   - Fetches controls from `GET /api/plugins/loxone/controls` on mount
   - Renders a table with columns: Name, Room, Type, Topic, Enabled (toggle)
   - Toggle calls `PUT /api/plugins/loxone/controls/:uuid` with `{ enabled: bool }`
   - Shows "Plugin not running" message if controls endpoint returns error
   - Shows control count in header
   - Uses Venus OS card styling consistent with other pages

6. **Create `webapp/js/pages/topic-routes.js`**: Preact component `TopicRoutes` that:
   - Fetches routes from `GET /api/plugins/loxone/routes` on mount
   - Renders existing routes as a list with: source topic, target topic, direction indicator, enabled toggle, delete button
   - "Add Route" form at bottom: source topic input, target topic input, direction dropdown (inbound/outbound), Add button
   - Save button calls `PUT /api/plugins/loxone/routes` with the full routes array
   - Uses Venus OS card styling

7. **Update api-client.js**: Add `fetchLoxoneControls()`, `toggleLoxoneControl(uuid, enabled)`, `fetchTopicRoutes()`, `saveTopicRoutes(routes)` functions.

8. **Update app.js routing**: Add routes for `#/loxone/controls` and `#/loxone/routes`. Add these as sub-items under the Loxone plugin entry in sidebar.

9. **Update sidebar.js**: When plugin id is "loxone" and status is "running", show sub-navigation links: "Controls" and "Topic Routes" indented under the Loxone plugin entry.
