```javascript
class LoxoneStructure {
  constructor(topicPrefix = 'loxone') // stores prefix, creates empty maps

  // Fetch LoxAPP3.json from Miniserver via HTTP
  async fetchStructure(host, port, user, pass) // GET http://user:pass@host:port/data/LoxAPP3.json

  // Build UUID-to-topic maps from parsed structure
  buildMap(loxApp3Json) // Iterates controls, rooms, cats. Maps each control UUID + sub-control state UUIDs.

  // Lookups
  uuidToTopic(uuid)     // returns topic string or undefined
  topicToUuid(topic)     // returns uuid string or undefined (control-level only, strips /state suffix)
  getMeta(uuid)          // returns { topic, name, type, room, uuid, category, stateKey? }
  getAll()               // returns array of all control-level mappings (not sub-states)

  // Internal
  slugify(str)           // lowercase, replace non-alphanumeric with hyphens, trim hyphens, empty -> "unknown"
}
```

Key implementation details:
- `rooms` object in LoxAPP3.json maps UUID -> { name, uuid, ... }
- `cats` (categories) object maps UUID -> { name, uuid, ... }
- `controls` object maps UUID -> { name, type, room (UUID ref), cat (UUID ref), states: { stateKey: stateUuid }, subControls: { uuid: controlObj } }
- For each control: look up room name from rooms map, category name from cats map
- Slugify room and control names for topic path segments
- Map sub-control states: each `states` entry maps `stateUuid -> {prefix}/{room}/{control}/{stateKey}`
- Also process `subControls` -- these are nested controls (e.g., a light controller has sub-controls for moods). Map them with their own names under the parent topic.
- Handle duplicate slugs in same room by appending `-{first8charsOfUuid}` to the second occurrence
- `fetchStructure` uses Node's global `fetch` (Node 20+) with basic auth via URL

  <name>Task 2: Loxone WebSocket client with binary protocol parser and reconnection</name>
  <files>plugins/loxone/loxone-ws.js, tests/loxone-ws.test.js</files>
  <behavior>
    - Connects to ws://host:port/ws/rfc6455 with Sec-WebSocket-Protocol: remotecontrol
    - HTTP basic auth via URL credentials (ws://user:pass@host/ws/rfc6455)
    - Emits "connected" event on successful WebSocket open
    - Sends "jdev/sps/enablebinstatusupdate" after connection to enable binary state events
    - Parses 8-byte binary message headers: byte 0=0x03, byte 1=identifier, bytes 4-7=payload length (LE uint32)
    - State machine: AWAITING_HEADER -> receives 8-byte header -> AWAITING_PAYLOAD -> receives payload -> processes -> AWAITING_HEADER
    - Value events (identifier=0x02): parses 24-byte chunks (16-byte UUID + 8-byte double), emits "valueEvent" with { uuid, value }
    - Text events (identifier=0x03): parses UUID + text, emits "textEvent" with { uuid, text }
    - Keepalive response (identifier=0x06): resets watchdog timer
    - Sends "keepalive" text every 60 seconds
    - Reconnects with exponential backoff (1s, 2s, 4s, 8s, 16s, cap 30s) + random jitter on close/error
    - Emits "disconnected" event on close
    - stop() cleanly closes connection and stops all timers
    - Text messages (non-binary) are emitted as "textMessage" events (JSON command responses)
  </behavior>
  <action>
Create `plugins/loxone/loxone-ws.js` exporting class `LoxoneWs` extending EventEmitter:

```javascript
class LoxoneWs extends EventEmitter {
  constructor({ host, port, user, pass })

  async connect()          // Opens WebSocket, sends enablebinstatusupdate, starts keepalive timer
  async disconnect()       // Closes WebSocket, stops timers, clears reconnect
  sendCommand(cmd)         // Sends text command (e.g., "jdev/sps/io/{uuid}/on")

  // Internal
  _onMessage(data, isBinary)   // Routes binary to state machine, text to textMessage event
  _processHeader(buffer)       // Parses 8-byte header, sets pending payload expectations
  _processPayload(buffer)      // Dispatches based on identifier type
  _parseValueEvents(buffer)    // Iterates 24-byte chunks, emits valueEvent per entry
  _parseTextEvents(buffer)     // Parses text event structure, emits textEvent
  _startKeepalive()            // setInterval 60s sending "keepalive"
  _scheduleReconnect()         // Exponential backoff with jitter
  _uuidFromBuffer(buf, offset) // Reads 16 bytes as formatted UUID string (8-4-4-4-12 with LE byte order for first 3 groups)
}
```

Key implementation details:
- Loxone UUID format in binary: first 3 groups are little-endian (4 bytes, 2 bytes, 2 bytes), last 2 groups are big-endian (2 bytes, 6 bytes). This is critical for correct UUID string generation.
- Use `ws` library directly: `new WebSocket(url, { headers: ... })` -- pass `Sec-WebSocket-Protocol: remotecontrol`
- Binary state machine tracks: `this._state` (HEADER or PAYLOAD), `this._pendingHeader` (parsed header info)
- Value event double: read as Float64 at each 24-byte chunk offset+16 using DataView with littleEndian=true
- Text events: 16-byte UUID + 4-byte padding (unused icon UUID ref) + 4-byte textLength (LE uint32) + UTF-8 text bytes
- Keepalive: if no keepalive response (identifier 0x06) within 15 seconds, treat connection as dead
- On reconnect success, emit "reconnected" event (distinct from initial "connected")
- `connect()` resolves when WebSocket opens or after 10s timeout (reject)
