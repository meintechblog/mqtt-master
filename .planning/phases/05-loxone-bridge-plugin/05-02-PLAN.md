```javascript
export default class LoxonePlugin {
  constructor()

  async start(context) {
    // 1. Read config: context.configService.get('plugins.loxone', {})
    // 2. Create LoxoneStructure(config.prefix || 'loxone')
    // 3. Fetch and build structure map
    // 4. Create LoxoneWs({ host, port, user, pass })
    // 5. Wire valueEvent -> MQTT publish
    // 6. Wire textEvent -> MQTT publish
    // 7. Subscribe to {prefix}/+/+/cmd for inbound commands
    // 8. Listen for MQTT messages matching cmd pattern -> sendCommand
    // 9. Connect WebSocket
    // 10. Publish HA Discovery configs if enableHaDiscovery is true
    // 11. Publish {prefix}/bridge/status = "online" (retained)
  }

  async stop() {
    // 1. Publish {prefix}/bridge/status = "offline" (retained)
    // 2. Disconnect WebSocket
    // 3. Unsubscribe from MQTT cmd topics
    // 4. Remove MQTT message listener
    // 5. Clear state
  }

  getStatus() {
    // { running, connected, controlCount, lastEvent (timestamp) }
  }

  getConfigSchema() {
    // JSON Schema with properties:
    // ip (string, title "Miniserver IP"), port (integer, default 80),
    // username (string), password (string, format "password"),
    // prefix (string, default "loxone"), autoStart (boolean, default false),
    // enableHaDiscovery (boolean, default true)
  }

  // Internal methods
  _onValueEvent(uuid, value)      // Lookup + publish
  _onTextEvent(uuid, text)        // Lookup + publish
  _onMqttMessage({ topic, payload }) // Pattern match + sendCommand
  _publishHaDiscovery()           // Iterate controls, map type -> HA component, publish config
  _isControlEnabled(uuid)         // Check disabledControls config
  _loxoneTypeToHaComponent(type)  // Type mapping table
}
```

Key implementation details:
- The MQTT message listener must filter for the plugin's prefix to avoid reacting to unrelated messages. Use `topic.startsWith(this.prefix + '/') && topic.endsWith('/cmd')`.
- For HA Discovery, each control publishes to `homeassistant/{component}/{nodeId}/{objectId}/config` where nodeId is `loxone_{miniserverSerial}` (or `loxone_bridge` if serial unavailable) and objectId is the slugified control name. Payload includes: `name`, `state_topic`, `command_topic` (for actuators), `unique_id`, `availability_topic`, `device` info.
- For value events, check `_isControlEnabled(uuid)` before publishing. If disabled, skip silently.
- Store `this._mqttHandler` reference to the message listener so it can be removed in `stop()`.
- On "reconnected" event from LoxoneWs, re-fetch structure (may have changed) and re-publish HA Discovery.
- Password field in config schema uses `format: "password"` -- the auto-generated form in Phase 4 renders text inputs, which is acceptable (the form already handles string type).
